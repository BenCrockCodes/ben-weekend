# NEOVOLT — a neon rhythm platformer

A browser-based rhythm platformer inspired by the feel of Geometry Dash, built
from scratch with **raw WebGL, ES6 modules, HTML and CSS** — no game engine,
no frameworks, no binary assets. All art is drawn by the renderer and all
audio (music included) is synthesized live with WebAudio.

## Running

The game uses ES modules and `fetch()`, so it must be served over HTTP:

```bash
npx serve .            # or: python -m http.server 5173
```

then open `http://localhost:5173`. Works in Chrome, Edge, Firefox and Safari.

**Controls:** Space / W / ↑ — the cube jumps (hold to keep jumping); the
ship rises while held and falls when released. Tap anywhere on mobile,
Esc/P to pause, R to restart.

## Menus & level ecosystem

The main menu is **Play · Character Select · Settings · Leaderboards**.
Play opens a Geometry Dash-style hub:

- **Play Main Levels** — the five official levels as animated cards with
  difficulty stars, song, best %, attempt counts, coins and lock state.
  Main levels are an *internal module* ([js/levelData.js](js/levelData.js),
  generated by the level tool) — nothing is fetched at runtime.
- **Play User Generated Levels** — the community screen; UI and a provider
  stub are in place, online servers are a future update.
- **My Created Levels** — every editor level with PLAYTEST / EDIT /
  SETTINGS, plus Create New. Playtesting returns here; editor test runs
  return to the editor.

Progression: each level records best %, attempts and completion; finishing
a level unlocks the next; the Leaderboards screen shows local stats
(global boards are future work).

## Gamemodes

- **Cube** — auto-run + jump (the original mechanics).
- **Ship** — hold to rise, release to fall (acceleration-based with a
  velocity cap, GD-style). The ship slides along floors and ceilings,
  dies on side hits and hazards, and tilts with its vertical speed.
  Gamemode portals (pink = ship, green = cube) switch modes mid-level and
  are available in the editor's "Gamemode Portals" tab.
- Levels 4 (**SKY DRIVE**, Normal) and 5 (**AFTERBURN**, Hard) are hybrid
  cube + ship levels; both were machine-verified completable (a corridor-
  following autopilot flies every ship section using the real physics).

The camera follows GD rules: its right edge stops exactly at the level's
end wall while the player keeps running until the finish.

## Online platform (Supabase)

The game is production-ready for **https://benfun.cc** with a Supabase
backend: email/password accounts, public player profiles with selectable
icons, cloud-synced save data (merged, never overwritten), profile wall
messages, and a `levels` table ready for user-generated uploads.

- All Supabase access goes through one module,
  [js/backend/backend.js](js/backend/backend.js); the database schema with
  Row Level Security on every table is [supabase/schema.sql](supabase/schema.sql).
- Connection settings live in
  [js/backend/backendConfig.js](js/backend/backendConfig.js) (publishable
  key only — the service key is never used).
- Unconfigured or offline builds run fully locally; the account screen
  explains how to go online.

**The complete production guide — Supabase project setup, database
creation, auth configuration, deploying to benfun.cc, and the launch
testing checklist — is in [SETUP.md](SETUP.md).**

## Newgrounds music import

Browsers can't call Newgrounds directly (no CORS headers), so the editor
imports songs through a tiny local proxy:

```bash
node tools/musicProxy.js     # http://localhost:8642
```

With the proxy running, enter a song ID in the editor's MUSIC panel and
IMPORT fetches the title, artist, duration, artwork and stream URL (tested
live with 467339 — "At the Speed of Light" by Dimrain47). Without it, the
editor explains exactly what to do; direct audio URLs always work.

## Level Editor

The main menu's **LEVEL EDITOR** opens a full in-game editor that shares the
game's renderer, objects and physics — anything you build is immediately
playable with **▶ TEST** (or **▶ HERE** to spawn at the camera).

- **Viewport**: wheel to zoom, middle-drag / hold-Space to pan, WASD to move,
  grid + snapping (G toggles the grid, snap and ½-block step in the toolbar).
- **Tools** (left): Select/Move (1), Place/Paint (2), Delete (3),
  Eyedropper (4), plus duplicate, flip H/V, rotate, align-row, undo/redo.
- **Palette** (bottom): every object in categorised tabs with SVG previews,
  instant search, Recent and ★ Favourites tabs.
- **Properties** (right): context-sensitive fields per object — size, count,
  portal mode, colour/opacity/rotation/layer for decorations.
- **Shortcuts**: Ctrl+C/V/X/D, Ctrl+Z/Y, Ctrl+S, Delete, arrows to nudge,
  right-click for a context menu.
- **Level settings**: name, description, creator, difficulty, BPM (speed is
  derived as `bpm / 15` to keep the beat grid), auto or manual length.
- **Colours**: live pickers (RGB/HSV/hex) for background, ground, accents
  and blocks.
- **Music**: pick a built-in soundtrack with preview, or import custom
  audio — Newgrounds IDs are attempted but blocked by their CORS policy
  (a clear error explains this) and a direct audio URL works today.
- **Saving**: levels persist to localStorage as the same JSON format the
  official levels use (`formatVersion: 2`), with export/import as .json
  files for sharing. Publish is a placeholder for the future online system.

Editor internals live in `js/editor/` (controller, object catalog, history,
storage, DOM panels); verified smooth with 4000+ placed objects.

## Design notes

- **Beat-locked levels.** Each level sets `speed = bpm / 15`, which makes one
  musical beat exactly 4 blocks. Obstacles sit on that grid, so the action
  lands on the soundtrack's rhythm.
- **Fixed timestep.** Physics always steps at 240 Hz regardless of display
  refresh rate, so the game feels identical at 60 or 240 fps.
- **Fairness.** Hazard hitboxes are smaller than their sprites, solid blocks
  have a snap tolerance for landings, and inputs are buffered ~100 ms. All
  three levels and all nine coins were verified completable by an automated
  solver driving the real physics engine.
- **Adding content.** New levels are new JSON files (add the id to
  `CONFIG.LEVEL_LIST`); new obstacle types need one hitbox builder in
  `levelManager.js` and one draw function in `gameObjects.js`. Future systems
  (character skins, achievements, more game modes) already have hooks in the
  save schema and module boundaries.
